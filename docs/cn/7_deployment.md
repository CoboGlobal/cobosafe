# 部署地址

为了节约部署 gas 与统一管理， Argus 中使用的大部分 `Cobo Safe` 相关合约使用不可升级的 [Minimal Proxy Contract](https://eips.ethereum.org/EIPS/eip-1167) 模式部署。合约 proxy 使用的 logic 代码由 `Cobo Factory` 合约维护。

`Cobo Factory` 已经部署在 6 条链上并使用相同的合约地址。
```
0xC0B00000e19D71fA50a9BB1fcaC2eC92fac9549C
```

> `Cobo Factory` 的 owner 权限由 Cobo 团队 3/5 多签维护。`Cobo Factory` 本身只提供索引作用，`Cobo Safe` 框架的运行本身不对 `Cobo Factory` 有任何依赖。对 `Cobo Factory` 上的合约更新不会直接影响任何已经创建过的 `Cobo Safe` 合约，对所有已创建的 `Cobo Safe` 合约的任何修改只能由其对应的 `Gnosis Safe` 进行操作。

任何均可通过 [pycobosafe](https://github.com/coboglobal/pycobosafe) 工具查看注册到 `Cobo Factory` 上的所有合约地址。
